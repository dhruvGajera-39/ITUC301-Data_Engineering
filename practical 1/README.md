# Practical 1 – Enterprise E-Commerce Data Engineering Lifecycle

## 📖 Overview

This repository contains the **Enterprise E-Commerce Data Engineering Lifecycle** diagram created using **Lucidchart** for **ITUC301 – Data Engineering Practical 1**.

The diagram represents the complete lifecycle of enterprise e-commerce data, from data generation and ingestion to storage, transformation, and serving, while incorporating security, data privacy, and observability throughout the pipeline.

---

## 🎯 Objective

To design a high-level data engineering architecture that demonstrates how enterprise data flows through different stages before being used for analytics and reporting.

---

## 🏗️ Workflow

### 1. Data Generation
- Website
- Mobile Application
- Orders
- Products

### 2. Data Ingestion
- API
- File Upload
- Data Validation

### 3. Data Storage
- MySQL Database
- Data Lake

### 4. Data Transformation
- Data Cleaning
- Remove Duplicate Records
- ETL Process

### 5. Data Serving
- Dashboards
- Reports
- Analytics

---

## 🔒 Cross-Cutting Components

The following components are integrated across the complete pipeline:

- **Security**
  - Authentication
  - Authorization
  - Data Encryption

- **Data Privacy**
  - Sensitive Data Protection
  - Compliance
  - Access Control

- **Observability**
  - Monitoring
  - Logging
  - Performance Tracking
